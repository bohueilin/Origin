## 3. Strategy note captured this session (for the pitch)

Why autonomous booking agents aren't in production yet — and our wedge:
- **Two bottlenecks, not one.** *Reliability* (long-horizon web tasks: τ-bench SOTA <50% success,
  pass^8 <25% — error compounding) is a **model** problem; we don't beat OpenAI there.
  *Trust/authorization/liability* (no way to bound, scope, revoke, or audit an agent holding your
  passport + card) is an **infrastructure gap** — that's our lane.
- OpenAI Operator (Jan 2025) → folded into "ChatGPT agent," standalone retired **Aug 31 2025**
  (officially "consolidation"; the reliability gap is benchmark-real but not their stated reason).
- The whole industry now agrees the missing layer is **authorization, not IQ**: Google **AP2**,
  Visa **Trusted Agent Protocol**, Mastercard **Agent Pay** all shipped in 2025. *"Permission is
  not the same as intent, and access is not the same as trust."*
- **Passport = the neutral, model-agnostic trust layer** (Auth0/Stripe/Plaid precedent).
