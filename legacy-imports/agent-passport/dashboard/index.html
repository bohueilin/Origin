How it works · three steps
1

Issue a passport

The authority mints a signed, time-boxed capability passport — scoped to one vendor, one action. The credential is fetched just-in-time from 1Password, in memory, never on disk.

2

Attenuate & verify

Every handoff can only narrow. Proof-of-possession binds the passport to the holder's key, so a stolen copy is inert. Each agent runs in its own isolated Daytona sandbox.

3

Kill & audit

One click revokes a compromised agent and cascades to every sub-agent — across the credential, compute, and identity planes. Every decision lands on a tamper-evident ledger.

Why not just OAuth scopes?

Four things plain tokens can't do.

Attenuation that only narrowsOAuth scopes can't shrink down a delegation chain; Okta's cross-app access forbids re-delegation. A Origin passport can only get smaller at each hop — least privilege by construction.
Proof-of-possessionA bearer token is replayable — steal it and you're in. A passport is bound to the holder's key (DPoP-style), so a stolen or replayed copy is inert.
Instant cascading killStandard OAuth can't revoke a live token — Auth0's default leaves a hijacked agent armed for 24h. We kill the capability mid-flight, and it cascades to every sub-agent.
Tamper-evident ledgerMutable SIEM logs aren't evidence. A hash-chained ledger makes every action provable and non-repudiable — exactly what the EU AI Act now requires.
