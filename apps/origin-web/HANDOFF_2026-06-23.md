## TL;DR

**Origin** turns a drawn factory floor into a deterministic safety gym and issues a
robot-readiness **license** (finish / escalate / refuse + RSL tier + FAR/FRR). A
deterministic oracle is the **only** judge. The product is a 4-step funnel:
**Submit your site → Build the robot brain → Run the proving ground → Get the license.**

Everything below the brief has shipped and is **live**. Recent work focused on the
"Submit your site" journey (the console at `/app.html`): a world-class site builder,
explicit multi-fleet deployment, proving-ground clarity, and a funnel-wide storytelling
+ palette pass.

## Non-negotiable invariants (do NOT break in any audit or refactor)

1. **The deterministic oracle is the ONLY judge.** No LLM-as-judge anywhere in the
   license path. `frozenToPlanInput` deliberately does NOT pass the site map; the
   single-agent BFS oracle (`siteEval.ts` / `bfsOracle`) mints the verdict.
2. **Honesty boundary.** A model number is "measured" ONLY if it came from a real
   oracle-scored run. Anything else is labeled **"projected"**. Never fabricate metrics.
3. **Multi-robot/multi-fleet is descriptive + animated only** — it never feeds the score.
4. **Secrets server-side only.** `.env*` is gitignored; only `VITE_*` is bundled. Never
   commit keys. `data/` (266 MB) and `factoryceo_trm/results/` are gitignored — never stage.
5. **Two `:root` palettes must stay in sync:** `src/App.css` (console) and
   `src/factorydad/factorydad.css` (marketing home).
